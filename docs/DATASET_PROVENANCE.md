# Dataset Provenance

This repository uses a synthetic enterprise-style corpus created for learning purpose.

The documents are not copied from a real employer, customer, or private system. They are original, public-safe examples inspired by common enterprise knowledge patterns:

- employee handbooks and HR operating policies
- IT support runbooks and incident procedures
- security policies and access-control standards
- compliance and audit evidence requirements
- finance planning reports and budget notes
- project briefs and operating updates

The corpus intentionally includes realistic imperfections:

- overlapping policy language across departments
- inconsistent terminology such as "remote work", "hybrid work", and "work from home"
- document-specific role and sensitivity metadata
- dated procedures and exception notes
- mixed source formats: Markdown, PDF, JSON logs, CSV records, and SQL dumps
- partial answers that require the assistant to refuse or ask for stronger evidence

The data is designed to test retrieval, citation quality, RBAC behavior, and grounded generation. It should be described as "synthetic enterprise-style data" in resumes and interviews, not as real company data.
